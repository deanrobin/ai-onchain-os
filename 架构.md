# AI On-Chain OS — 系统架构文档

> 版本：2026-03 | 技术栈：Spring Boot 3.4.3 / JDK 17 / MySQL 8.0 / Thymeleaf / Bootstrap 5

---

## 一、整体架构概览

```
┌──────────────────────────────────────────────────────────────┐
│                         OKX Web3 API                         │
│    (信号列表 / 钱包画像 / 交易记录 / 代币余额 / 行情价格)         │
└──────────────┬───────────────────────────────────────────────┘
               │ HTTP (签名鉴权)
               ▼
┌──────────────────────────────────────────────────────────────┐
│                  Spring Boot Dashboard 应用                   │
│  ┌────────────────┐  ┌─────────────────┐  ┌───────────────┐ │
│  │ SignalFetchJob │  │ WalletAnalyzeJob│  │ PriceFetchJob │ │
│  │  (10s/次)      │  │  (5min/次)      │  │  (30s/次)     │ │
│  └───────┬────────┘  └────────┬────────┘  └──────┬────────┘ │
│          │                    │                   │          │
│          ▼                    ▼                   ▼          │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │                      MySQL 数据库 (oc_os)                │ │
│  │  smart_money_signal │ smart_money_wallet │ price_ticker  │ │
│  │  wallet_tx_cache    │ my_address                        │ │
│  └──────────────────────────────┬──────────────────────────┘ │
│                                 │                            │
│  ┌──────────────────────────────▼──────────────────────────┐ │
│  │        Controller层 (页面 + REST API)                    │ │
│  │    DashboardController    ApiController                  │ │
│  └──────────────────────────────┬──────────────────────────┘ │
└─────────────────────────────────┼────────────────────────────┘
                                  │ HTTP :9900
                                  ▼
               ┌──────────────────────────────────┐
               │       浏览器 (用户 Dashboard)      │
               │  Thymeleaf SSR + AJAX 定时刷新    │
               └──────────────────────────────────┘
```

**核心原则：Web 请求永远只读 DB，OKX API 只由后台 Job 调用，保证响应速度与限流安全。**

## 二、数据库结构 (DB: `oc_os`)

### 2.1 `smart_money_signal` — 聪明钱买入信号

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 自增主键 |
| chain_index | varchar(10) | 链标识：`56`=BSC, `501`=Solana |
| token_address | varchar(100) | 代币合约地址 |
| token_symbol | varchar(50) | 代币符号，如 `PEPE` |
| token_name | varchar(200) | 代币全名 |
| token_logo | varchar(500) | Logo URL |
| wallet_type | varchar(30) | 钱包类型，`1`=聪明钱 |
| trigger_wallet_count | int | 触发信号的钱包数量 |
| trigger_wallets | text | 钱包地址列表（逗号分隔） |
| amount_usd | decimal(20,4) | 买入总金额（USD） |
| price_at_signal | decimal(30,12) | 信号触发时代币价格 |
| market_cap_usd | decimal(20,4) | 代币市值（USD） |
| sold_ratio_percent | decimal(12,4) | 聪明钱已卖出比例（0-100） |
| signal_time | datetime | 信号时间（Asia/Shanghai） |
| created_at | datetime | 记录创建时间 |

**过滤规则（双重）：**
- `market_cap_usd < $10,000` → 丢弃（市值过小，数据不可信）
- `amount_usd / market_cap_usd > 50%` → 丢弃（买入额超市值一半，数据异常）

**去重规则：**
- 2 分钟窗口内，同 token 同链：若 `amount_usd` 或 `trigger_wallet_count` 增长 → UPDATE 更新，否则跳过
- 超过 2 分钟 → INSERT 新记录

---

### 2.2 `smart_money_wallet` — 聪明钱钱包档案

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 自增主键 |
| address | varchar(100) | 钱包地址 |
| chain_index | varchar(10) | 链标识 |
| label | varchar(200) | 标签（可选） |
| score | decimal(10,4) | 综合评分（0-100） |
| win_rate | decimal(10,4) | 胜率（0-1，如 0.72 = 72%） |
| realized_pnl_usd | decimal(20,4) | 已实现盈亏（USD） |
| buy_tx_count | int | 买入次数 |
| sell_tx_count | int | 卖出次数 |
| avg_buy_value_usd | decimal(20,4) | 平均买入金额 |
| source | varchar(50) | 来源，固定 `signal`（从信号中提取） |
| last_analyzed_at | datetime | 上次被 WalletAnalyzeJob 分析时间 |
| overview_json | text | OKX 画像原始 JSON 缓存 |
| overview_updated_at | datetime | 画像缓存更新时间 |
| tx_count | int | 交易总数 |
| created_at / updated_at | datetime | 创建/更新时间 |

**评分公式（WalletScorerService）：**
```
score = winRate × 35
      + pnlLogScale × 30   (PnL 对数归一化，上限 10 万 USD)
      + activity × 20      (买卖次数活跃度，上限 50 次)
      + consistency × 15   (胜率 × 活跃度一致性加成)
      + bonus × 5          (胜率 > 70% 且 PnL > 0 双重达标奖励)
```

### 2.3 `wallet_tx_cache` — 钱包交易记录缓存

| 字段 | 类型 | 说明 |
|------|------|------|

| address | varchar(100) | 钱包地址 |
| chain_index | varchar(10) | 链标识 |
| tx_hash | varchar(150) | 交易哈希 |

## 三、OKX API 接入

所有 API 通过 `OkxApiClient` 统一调用，支持 HMAC-SHA256 签名（`OKX_API_KEY` / `OKX_API_SECRET` / `OKX_PASSPHRASE` 从环境变量读取）。

| API | 路径 | 用途 | 调用方 |
|-----|------|------|--------|
| **信号列表** | `POST /api/v6/dex/market/signal/list` | 获取聪明钱实时买入信号 | SignalFetchJob |
| **钱包画像** | `GET /api/v6/dex/market/portfolio/overview` | 获取钱包胜率/PnL/均价等综合画像 | WalletAnalyzeJob / WalletCacheService |
| **近期盈亏** | `GET /api/v6/dex/market/portfolio/recent-pnl` | 获取近期代币盈亏明细 | PortfolioService（我的持仓页） |
| **交易记录** | `GET /api/v6/dex/post-transaction/transactions-by-address` | 获取钱包链上交易历史 | WalletCacheService |
| **代币余额** | `GET /api/v5/wallet/asset/token-balances-by-address` | 获取钱包代币持仓 | PortfolioService |
| **行情价格** | `GET /api/v5/market/ticker?instId=BTC-USDT` | 获取 BTC/ETH/BNB/SOL 现货价格 | PriceFetchJob（无需签名） |

**限流保护：**
- 遇到 HTTP 429 → 自动退避重试（5s / 10s / 15s，最多 3 次）
- WalletAnalyzeJob 每批 20 个钱包，批间 500ms 延迟
- 链间请求延迟 2s，避免集中打爆

## 四、后台 Job 机制

### 4.1 SignalFetchJob — 信号采集（10 秒固定延迟）

```
每 10 秒执行：
  for chain in [56, 501]:
    POST /api/v6/dex/market/signal/list
      { chainIndex, walletType=1, minAmountUsd=5000, limit=20 }
    
    for each signal:
      提取 triggerWalletAddress → 写入 smart_money_wallet（新地址）
      
      双重过滤：
        ① marketCapUsd < 10K → skip
        ② amountUsd / marketCapUsd > 50% → skip（数据异常）
      
      去重逻辑：
        findLatest(chain, tokenAddr, walletType)
        ├── 2min 内存在 & 数据无变化 → skip
        ├── 2min 内存在 & amt/count 增长 → UPDATE 现有记录
        └── 超过 2min 或全新 token → INSERT 新记录
```

### 4.2 WalletAnalyzeJob — 钱包评分（5 分钟间隔）

```
每 5 分钟执行：
  查询 smart_money_wallet（未在 15min 内分析过的，按 score ASC 取 20 个）
  
  for each wallet:
    GET /api/v6/dex/market/portfolio/overview
      { chainIndex, walletAddress, timeFrame=3(7天) }
    
    解析：winRate / realizedPnlUsd / buyTxCount / avgBuyValueUsd...
    调用 WalletScorerService 计算综合评分
    UPDATE smart_money_wallet SET score=?, win_rate=?, ...
```

### 4.3 PriceFetchJob — 主流币价格（30 秒间隔）

```
每 30 秒执行：
  for symbol in [BTC, ETH, BNB, SOL]:
    GET /api/v5/market/ticker?instId={symbol}-USDT
    解析 last(最新价) + change24h
    UPSERT price_ticker
```

## 五、Web 接口（REST API）

所有 API 前缀 `/api`，端口 `9900`。

| 接口 | 方法 | 说明 | 数据来源 |
|------|------|------|------|
| `/api/prices` | GET | BTC/ETH/BNB/SOL 最新价格 | DB `price_ticker` |
| `/api/signals/db` | GET | 最近信号列表（过滤市值/比例异常），`?limit=N` | DB `smart_money_signal` |
| `/api/signals` | GET | OKX 实时信号（直接调 OKX，仅聪明钱页用） | OKX API |
| `/api/wallets` | GET | 聪明钱排行榜，`?limit=N` | DB `smart_money_wallet` |
| `/api/wallet/overview` | GET | 单钱包画像（走缓存层），`?address=&chain=&timeFrame=` | DB/OKX |
| `/api/portfolio/overview` | GET | 用户持仓地址画像 | DB/OKX |
| `/api/portfolio/pnl` | GET | 用户持仓近期盈亏 | OKX API |
| `/api/portfolio/txs` | GET | 用户持仓交易记录 | OKX API |

## 七、整体数据流转

```
                         ┌─────────────────┐
                         │   OKX Web3 API  │
                         └────────┬────────┘
                                  │
          ┌───────────────────────┼──────────────────────┐
          │                       │                      │
     每10秒                   每5分钟                 每30秒
          │                       │                      │
   SignalFetchJob         WalletAnalyzeJob          PriceFetchJob
          │                       │                      │
    ①写信号 +              ②更新评分/画像           ③写价格
    提取钱包地址                   │                      │
          │                       │                      │
          └──────┬────────────────┘                      │
                 ▼                                        ▼
        smart_money_signal              price_ticker (BTC/ETH/BNB/SOL)
        smart_money_wallet
                 │
                 │ Web 请求触发（用户点击详情）
                 ▼
         WalletCacheService
         ┌───────────────────────────────────────┐
         │  overview_json 有效(TTL 300s)? → 读DB │
         │  否 → 调 OKX → 写 wallet_tx_cache     │
         │       + 更新 overview_json             │
         └───────────────────────────────────────┘
                 │
                 ▼
          DashboardController → Thymeleaf 渲染
                 │
                 ▼
           浏览器展示 (light 主题)
           ├── AJAX 10s/30s/60s 定时刷新（不重载页面）
           ├── 点代币 → 跳 GMGN / OKX 行情页
           └── 复制地址 → clipboard fallback（HTTP 兼容）
```

## 八、关键配置

**应用配置（`application.yml`）：**
```yaml
smart-money:
  jobs:
    signal-fetch:
      chains: "56,501"           # BSC + Solana
      min-amount-usd: "5000"     # 买入门槛 $5000
      min-market-cap-usd: 10000  # 市值门槛 $10K
    wallet-analyze:
      interval-minutes: 5        # 每 5 分钟分析一批
      max-wallets: 20            # 每批上限
  cache:
    overview-ttl-seconds: 300    # 画像缓存 5 分钟
    tx-history-ttl-seconds: 30  # 交易缓存 30 秒
```

**日志（Log4j2）：**
- 路径：`/root/aios-dashboard/logs/`
- `info.log`：INFO + WARN + ERROR 全量
- `error.log`：ERROR 单独
- 滚动：超 100MB 或每日切割，压缩归档，7 天自动清理

**环境变量（`/etc/environment`）：**
```
OKX_API_KEY=...
OKX_API_SECRET=...
OKX_PASSPHRASE=...
```
